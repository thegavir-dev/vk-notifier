## Как это работает

Плагин подключается к стандартному фильтру WordPress `wp_mail` и перехватывает все исходящие письма сайта до их отправки. Тема и тело письма очищаются от HTML, формируется текстовое сообщение и отправляется в VK через официальный API (`messages.send`) — в личные сообщения пользователям или в беседы сообщества.

Оригинальное письмо при этом не блокируется и доставляется получателю как обычно.

---

## Установка

1. Скопировать папку `vk-notifier` в `wp-content/plugins/`
2. Активировать плагин через меню **Плагины** в WordPress
3. Открыть **Настройки → VK Notifier**
4. Заполнить токен VK и ID получателей
5. Нажать **«Отправить тестовое сообщение»** для проверки

## Настройка

| Поле | Описание |
|---|---|
| **Токен VK** | Токен доступа сообщества или пользователя. Требуется разрешение `messages`. |
| **ID сообщества VK** | Числовой ID сообщества (положительное число). Обязателен при использовании токена сообщества. |
| **Получатели VK** | ID пользователей или `peer_id` бесед через запятую. Только положительные числа. |
| **Пересылать** | Режим пересылки: все письма или только из белого списка. |
| **Белый список email** | Email-адреса через запятую. Используется только в режиме «Только письма из белого списка». |
| **Префикс сообщения** | Текст, добавляемый в начало каждого сообщения VK. |
| **Текст о вложениях** | Добавляется в конец сообщения, если в письме были вложения. |
| **Уровень логирования** | Что записывать в журнал: все события, только ошибки, только успешные, только информационные. |
| **Хранить логи (дней)** | Срок хранения записей журнала. По умолчанию `30`. Значение `0` отключает автоматическую ротацию. |
| **Режим отправки** | Мгновенно или через WP-Cron. |

## Получение токена VK

### Токен сообщества (рекомендуется)

1. Открыть страницу своего сообщества ВКонтакте
2. Перейти в **Управление → Работа с API → Ключи доступа**
3. Нажать **«Создать ключ»**
4. Поставить галочку **Сообщения** → **Разрешить**
5. Скопировать полученный токен
6. В настройках плагина указать токен и числовой ID сообщества (без минуса)

> **Важно.** ID сообщества в поле настроек — это положительное число (например, `123456789`). Отрицательные значения не поддерживаются.

### Токен пользователя

Для личного токена поле **ID сообщества** оставить пустым. Токен пользователя можно получить через [VK API](https://vk.com/dev/access_token), предоставив разрешение `messages`.

## Режимы пересылки

**Только письма из белого списка** (по умолчанию)
Письмо пересылается в VK, только если адрес получателя (`To:`) совпадает с одним из email в белом списке. Если белый список пуст — ничего не пересылается.

**Все письма**
Все перехваченные письма сайта дублируются в VK без проверки адресата. Белый список при этом игнорируется.

> Поле «Белый список email» автоматически скрывается в настройках при выборе режима «Все письма».

## Поиск peer_id беседы

`peer_id` беседы — это числовой идентификатор вида `2000000XXX`. Он отличается от обычного `chat_id`.

Чтобы получить корректный `peer_id`:

1. Открыть **Настройки → VK Notifier**
2. Убедиться, что токен и ID сообщества заполнены
3. Нажать **«Найти доступные беседы сообщества»**
4. В появившейся таблице скопировать значение из колонки **peer_id**
5. Вставить это значение в поле **Получатели VK**

> Использование `chat_id` вместо `peer_id` или отрицательного ID приведёт к ошибке отправки. Плагин намеренно блокирует отрицательные значения.

## Журнал событий

Журнал доступен по адресу **Настройки → VK Notifier Logs**.

Каждая запись содержит:
- дату и время события
- уровень: `success`, `error`, `info`
- тип события: `messages.send`, `validation`, `find_conversations`, `email_intercept`
- текст сообщения и детали (recipient, код ошибки VK API и т.д.)

Доступна фильтрация по уровню, типу события и поиск по тексту. Кнопка **«Очистить логи»** удаляет все записи.

Автоматическая ротация настраивается через поле **«Хранить логи (дней)»**. Очистка запускается ежедневно через WP-Cron.

---

## Режимы отправки

**Сразу** (по умолчанию)
Сообщение отправляется в VK в момент перехвата письма, в том же запросе WordPress. Подходит для большинства сайтов.

**Через WP-Cron**
Отправка ставится в очередь и выполняется фоново при следующем запуске WP-Cron. Полезно на сайтах, где запросы к внешним API в потоке страницы нежелательны. На сайтах с нестабильным или отключённым WP-Cron уведомления могут задерживаться — в этом случае рекомендуется режим «Сразу».

Статус WP-Cron отображается прямо на странице настроек.

## Хуки и интеграции

| Хук | Тип | Описание |
|---|---|---|
| `wp_mail` | filter | Перехват исходящих писем |
| `messages.send` | VK API | Отправка сообщений |
| `vk_notifier_cleanup_logs` | WP-Cron | Ежедневная ротация логов |
| `vk_notifier_async_send` | WP-Cron | Фоновая отправка в режиме «Через WP-Cron» |

## Ограничения

- Поддерживаются только положительные `user_id` и `peer_id`. Отрицательные ID получателей блокируются на уровне валидации.
- Длина сообщения VK ограничена 4096 символами. Если письмо длиннее — текст обрезается с добавлением `…`.
- Вложения из писем не передаются в VK. Если в письме есть вложения, в сообщение добавляется настраиваемый текст-уведомление.
- При режиме пересылки «Все письма» плагин дублирует в том числе системные уведомления WordPress (сброс пароля, регистрация пользователя и т.д.). Рекомендуется использовать белый список, если это нежелательно.
- Для токена сообщества бот должен иметь возможность писать пользователю — пользователь должен быть подписчиком сообщества или начать диалог первым.

## Деинсталляция

При удалении плагина через **Плагины → Удалить** выполняется полная очистка:

- удаляется таблица `{prefix}vk_notifier_logs`
- удаляются опции `vk_notifier_options` и `vk_notifier_db_version`
- снимается WP-Cron событие `vk_notifier_cleanup_logs`
